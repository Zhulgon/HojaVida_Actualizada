Prediccion con datos sinteticos

Perdidas logisticas y comparacion de modelos.

Este ejercicio construye un dataset sintetico sobre envios y perdidas de unidades, con variables como cantidad enviada, distancia, modo de transporte y fragilidad. Luego explora relaciones, entrena una regresion lineal y compara su desempeno contra un arbol de decision.

1000 muestras Dataset sintetico generado con semilla controlada.
4 features Cantidad, distancia, modo de transporte y fragilidad.
MAE / MSE Metricas usadas para comparar calidad de prediccion.
2 modelos Regresion lineal y arbol de decision como contraste.

Valor del caso

  • No se queda en entrenar: hace EDA, separa train/test y compara enfoques.
  • Relaciona variables operativas con una salida cuantitativa.
  • Incluye una lectura critica de por que un modelo simple puede ser mejor.
pandas LinearRegression DecisionTreeRegressor

Datos

Como se construye el problema.

La variable objetivo se genera a partir de una formula con ruido. Eso hace que el caso sea ideal para observar cuando la relacion subyacente favorece a un modelo lineal.

Cantidad enviada

Representa el volumen base del envio y empuja parte de la perdida esperada.

Distancia y fragilidad

Agregan complejidad operativa y explican por que ciertos despachos pierden mas unidades.

Modo de transporte

Introduce una variable discreta con efecto directo sobre el resultado final.

Modelado

Regresion lineal primero, arbol despues.

El notebook hace algo muy sano: no asume que el segundo modelo sera mejor solo por ser mas complejo.

Regresion lineal

Entrena sobre el conjunto de entrenamiento, revisa coeficientes e intercepto y luego calcula MAE y MSE sobre test.

Arbol de decision

Usa DecisionTreeRegressor(max_depth=5) y compara sus metricas con la linea base lineal.

El propio notebook concluye que la regresion lineal predice mejor en este caso, lo cual tiene sentido porque los datos sinteticos fueron generados con una relacion basicamente lineal.

Lectura final

Que comunica este caso.

Muestra capacidad para estructurar un problema de prediccion, explorar datos, elegir metricas y justificar por que un modelo funciona mejor que otro.

EDA antes de modelar

Incluye dispersion, boxplot y barplot para inspeccionar relaciones entre variables.

Comparacion honesta

No vende el modelo mas complejo como ganador automatico; deja que las metricas hablen.

Enfoque aplicado

El caso esta bien alineado con operaciones, logistica y analitica de negocio.
