CNN multiclase

CIFAR-10 filtrado para tres clases concretas.

Este ejercicio toma CIFAR-10 y lo reduce a tres categorías: automobile, ship y truck. A partir de ahi normaliza imágenes, construye un conjunto de validación, entrena una CNN pequeña y evalúa tanto métricas como predicciones visuales sobre imágenes del test.

3 clases Automobile, ship y truck remapeadas a 0, 1 y 2.
20% validación Separación explicita del entrenamiento para controlar aprendizaje.
10 epocas Entrenamiento inicial con batch_size=32.
Softmax Salida multiclase con tres probabilidades posibles.

Resumen del caso

  • No usa todo CIFAR-10, sino una version filtrada y controlada.
  • Incluye un split de validación separado del test final.
  • Combina métricas con ejemplos visuales de aciertos y errores.

Pipeline

Desde el dataset completo hasta tres clases utiles.

El valor del ejercicio esta en saber acotar el problema y preparar bien los datos antes de entrenar.

1

Se cargan las imágenes de CIFAR-10 y se aplanan las etiquetas para trabajar con más comodidad.

2

Se filtran solo las clases [1, 8, 9] y luego se remapean a 0, 1, 2.

3

Las imágenes se normalizan y se crea un conjunto de validación con train_test_split.

4

Se entrena la CNN y luego se grafican accuracy, loss y ejemplos del set de prueba.

Modelo

Una CNN pequeña, clara y bien enfocada.

La arquitectura combina dos bloques convolucionales con pooling, una capa densa y una salida softmax. Es un modelo razonable para un ejercicio de práctica en clasificación multiclase.

Arquitectura

Conv2D(32) -> MaxPooling -> Conv2D(64) -> MaxPooling -> Flatten -> Dense(64) -> Dense(3, softmax)

Resultados que se observan

El cuaderno compara curvas de entrenamiento y validación, y además presenta imágenes aleatorias con su clase real y predicha para revisar el comportamiento del modelo más alla del accuracy.

Lectura

Que comunica este ejercicio.

Este caso muestra que no solo sabes usar una CNN, también sabes recortar un problema, construir validación y revisar aprendizaje con graficas y ejemplos.

Preparación de datos

Filtrado de clases, remapeo de etiquetas y normalizacion correcta de imagenes.

Entrenamiento ordenado

Uso explicito de validación para seguir la evolución del modelo durante las epocas.

Interpretación visual

Comparación entre predicción y realidad para detectar si el modelo generaliza de forma creible.
