Clasificador CNN de números manuscritos.

Este caso parte de un ejercicio en Python trabajado en Colab para entrenar una red neuronal convolucional sobre MNIST. El flujo incluye preparación de imágenes, codificación de etiquetas, construcción del modelo, entrenamiento, evaluación y lectura visual de predicciones.

Aunque el archivo se llama clasificadorNumeros1-9, la implementación principal clasifica diez clases reales del dataset: del 0 al 9. Dentro del mismo archivo también dejé una extensión con CIFAR-10 para seguir explorando clasificación multiclase en imágenes a color.

Desarrollo del caso

Flujo de trabajo del clasificador.

La estructura del ejercicio sigue una secuencia clara de machine learning para imágenes: cargar, preparar, entrenar, evaluar e interpretar. Aquí se ve el paso a paso sin perder el sentido de lo que hace cada bloque.

1

Carga de datos

Se importa MNIST desde TensorFlow/Keras y se separa en entrenamiento y prueba.

2

Preprocesamiento

Las imágenes se redimensionan a 28 x 28 x 1 y se normalizan a valores entre 0 y 1.

3

Etiquetas

Las clases pasan a formato one-hot con to_categorical para usar softmax.

4

Entrenamiento

La CNN se ajusta con Adam, pérdida categórica y un 10% del train para validación.

5

Lectura de resultados

Se evalúa en test y luego se comparan etiquetas reales vs. predichas para interpretar el modelo.

Arquitectura

Cómo está construida la CNN.

La red combina capas convolucionales para extraer patrones visuales y capas densas para convertir esos rasgos en una decisión final. Es un flujo clásico pero muy útil para entender deep learning aplicado a imágenes.

Input Imagen 28 x 28 x 1 en escala de grises.
→
Conv2D · 32 filtros Detecta bordes, trazos y rasgos simples del dígito.
MaxPooling Reduce dimensionalidad y conserva los patrones más relevantes.
Conv2D · 64 filtros Aprende patrones visuales más complejos combinando rasgos previos.
Dense · 64 Resume las características y prepara la clasificación final.
Softmax · 10 Entrega una probabilidad por cada clase del 0 al 9.

Lo que resuelve cada capa

Las convoluciones encuentran patrones espaciales; el pooling reduce ruido y costo computacional; las capas densas transforman esas características en una predicción interpretable.

Por qué softmax

No es un problema binario. Hay varias clases posibles, por eso la salida correcta es una distribución de probabilidad sobre diez opciones.

Lo importante del ejercicio

Más que repetir una receta, este caso demuestra comprensión de preprocesamiento, estructura de red, entrenamiento inicial y lectura de resultados en clasificación de imágenes.

Interpretación

Cómo leer las predicciones del modelo.

El script evalúa el conjunto de prueba y luego muestra ejemplos reales junto a la predicción. El punto clave es entender que la red entrega probabilidades, no solo un número aislado.

Lectura del output

En el flujo de prueba se usa np.argmax(predictions[i]) para escoger la clase con mayor probabilidad. Esa es la salida final que se compara con la etiqueta real.

Bloques clave

Código principal del ejercicio.

Estos fragmentos reflejan exactamente la estructura central del archivo trabajado en Colab para el clasificador de números.

Carga y preparación de MNIST

from tensorflow.keras.datasets import mnist
from tensorflow.keras.utils import to_categorical

(x_train, y_train), (x_test, y_test) = mnist.load_data()

x_train = x_train.reshape((x_train.shape[0], 28, 28, 1)).astype("float32") / 255
x_test = x_test.reshape((x_test.shape[0], 28, 28, 1)).astype("float32") / 255

y_train = to_categorical(y_train)
y_test = to_categorical(y_test)

El preprocesamiento adapta el dataset a una CNN y normaliza los pixeles para entrenar con mayor estabilidad.

Modelo convolucional

model = Sequential()
model.add(Conv2D(32, (3, 3), activation="relu", input_shape=(28, 28, 1)))
model.add(MaxPooling2D((2, 2)))
model.add(Conv2D(64, (3, 3), activation="relu"))
model.add(MaxPooling2D((2, 2)))
model.add(Flatten())
model.add(Dense(64, activation="relu"))
model.add(Dense(10, activation="softmax"))

La primera parte extrae patrones visuales y la salida softmax convierte el resultado en probabilidades por clase.

Entrenamiento y evaluación

model.compile(
    optimizer="adam",
    loss="categorical_crossentropy",
    metrics=["accuracy"]
)

model.fit(x_train, y_train, epochs=5, batch_size=64, validation_split=0.1)

test_loss, test_acc = model.evaluate(x_test, y_test)
print(f"Precisión en test: {test_acc:.4f}")

Se usa una configuración inicial sólida para un ejercicio académico y se reserva parte del entrenamiento para validación.

Predicción visual

predictions = model.predict(x_test)

for i in range(10):
    plt.imshow(x_test[i].reshape(28, 28), cmap="gray")
    plt.title(
        f"Real: {np.argmax(y_test[i])}, "
        f"Predicho: {np.argmax(predictions[i])}"
    )
    plt.axis("off")
    plt.show()

Esta lectura aterriza el comportamiento del modelo con ejemplos concretos del conjunto de prueba.
